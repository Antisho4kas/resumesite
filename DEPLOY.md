# 🚀 Развертывание на GitHub Pages

## 📋 Пошаговая инструкция

### 1. Форкните репозиторий
1. Перейдите на страницу репозитория
2. Нажмите кнопку "Fork" в правом верхнем углу
3. Выберите свой аккаунт

### 2. Клонируйте ваш форк
```bash
git clone https://github.com/ВАШ_ЮЗЕРНЕЙМ/resume-website.git
cd resume-website
```

### 3. Кастомизируйте сайт
Откройте файлы в редакторе и измените:

#### Контактные данные:
- `email@example.com` → ваш email
- `+49 1782673024` → ваш телефон  
- `Munchen, Germany` → ваше местоположение

#### Социальные сети:
- `https://linkedin.com/in/yourprofile` → ваш LinkedIn
- `https://github.com/yourusername` → ваш GitHub
- `https://t.me/yourusername` → ваш Telegram
- `https://twitter.com/yourusername` → ваш Twitter

#### Файлы для скачивания:
В `files.html` добавьте ссылки на ваши реальные файлы

### 4. Загрузите изменения
```bash
git add .
git commit -m "Customize portfolio with my data"
git push origin main
```

### 5. Включите GitHub Pages
1. Перейдите в ваш репозиторий на GitHub
2. Нажмите "Settings" (⚙️)
3. В меню слева выберите "Pages"
4. В разделе "Build and deployment":
   - Source: Deploy from a branch
   - Branch: main
   - Folder: /root
5. Нажмите "Save"

### 6. Дождитесь развертывания
- GitHub автоматически развернет сайт
- Через 1-5 минут сайт будет доступен
- Статус можно посмотреть в Actions → deployments

## 🌐 URL вашего сайта

Ваш сайт будет доступен по адресу:
```
https://ВАШ_ЮЗЕРНЕЙМ.github.io/resume-website
```

## ✅ Проверка

1. **Откройте сайт** по вашему URL
2. **Проверьте адаптивность** - откройте на мобильном устройстве
3. **Проверьте языковой переключатель** - должен работать
4. **Проверьте все ссылки** - должны вести на нужные страницы
5. **Проверьте форму контактов** - должна отправляться

## 🔧 Если что-то не работает

### Сайт не отображается:
1. Убедитесь, что GitHub Pages включен (Settings → Pages)
2. Проверьте, что вы загрузили файлы в ветку `main`
3. Посмотрите логи в Actions → deployments

### Стили не загружаются:
1. Проверьте, что файл `styles.css` существует
2. Откройте консоль браузера (F12) на наличие ошибок
3. Убедитесь, что пути к файлам правильные

### Языковой переключатель не работает:
1. Проверьте консоль браузера на ошибки JavaScript
2. Убедитесь, что все элементы имеют `data-i18n` атрибуты
3. Проверьте localStorage в браузере

## 🎨 Дальнейшая кастомизация

### Изменение цветов:
В `styles.css` найдите:
```css
:root {
  --primary-color: #2563eb;     /* Основной цвет */
  --secondary-color: #0ea5e9;   /* Дополнительный */
}
```

### Добавление новых страниц:
1. Создайте новый HTML файл по аналогии с существующими
2. Добавьте ссылку в навигацию во всех файлах
3. Обновите `sitemap.xml`

### Добавление новых языков:
1. В `script.js` добавьте переводы в объект `translations`
2. Добавьте кнопку языка в HTML
3. Обновите README
